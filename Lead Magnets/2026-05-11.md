# The Teams Rooms Health Checklist: 15 Checks to Run Before Your Next All-Hands

# The Teams Rooms Health Checklist
## 15 Checks to Run Before Your Next All-Hands
*For IT Managers, AV Integrators & MSPs | ~12 minutes to complete*

---

### Why This Exists
The average mid-market company has 8–20 Teams Rooms deployed. Studies show **42% of meeting room technology fails at least once per month** — usually discovered live, in front of executives or clients. This checklist catches the 15 most common failure points before they cost you.

**Estimated time saved per incident avoided: 45–90 minutes of firefighting + reputational damage.**

---

### PRE-MEETING: Device & Hardware (Do 24 hrs before)

- [ ] **1. Camera firmware is current**
  - Check: Teams Admin Center → Devices → Cameras → Firmware version
  - Target: Within 1 version of latest release
  - ⚠️ Outdated firmware = #1 cause of video freezing

- [ ] **2. Compute device (MTR) is on supported OS build**
  - Check: Teams Admin Center → Devices → Teams Rooms on Windows → OS version
  - Minimum: Windows 11 22H2 or later
  - Fix if failing: Push update via Intune or TAC remote action

- [ ] **3. Teams Rooms app version is current**
  - Target: Within 2 releases of latest (check aka.ms/roomsapp)
  - Outdated app = missing features AND known bugs

- [ ] **4. Audio peripheral (speaker/mic bar) is recognized**
  - Check: On-device settings → Peripherals — all show 'Connected'
  - Test: Run test call from the room console
  - Common miss: USB devices silently disconnect after Windows update

- [ ] **5. Display(s) power on automatically with room session**
  - Test: Start a test meeting, confirm all screens wake within 10 seconds
  - Fix: Check HDMI-CEC settings on display AND MTR config XML

---

### NETWORK & CONNECTIVITY (Do same day)

- [ ] **6. Room network is on dedicated VLAN or QoS-tagged segment**
  - Why: Consumer-grade flat networks cause audio packet loss above 50 concurrent users
  - Check with: Network admin or run `ping -t` to Teams relay IPs — look for >20ms jitter

- [ ] **7. Required Microsoft 365 URLs/IPs are whitelisted**
  - Reference: aka.ms/o365endpoints (Category: Optimize + Allow)
  - Common miss: UDP ports 3478–3481 blocked = forced TCP = degraded call quality

- [ ] **8. Bandwidth available meets minimums**
  - Minimum per room: 4 Mbps up/down for 1080p video
  - For large rooms with content sharing: 8–10 Mbps recommended
  - Test: speedtest.net from the MTR device browser or use Network Assessment Tool

- [ ] **9. Room resource account is licensed and sign-in is healthy**
  - Check: Teams Admin Center → Devices — account shows 'Signed in'
  - License required: Teams Rooms Basic (free, up to 25 rooms) or Teams Rooms Pro
  - ⚠️ Expired password = silent sign-out = room appears 'offline' with no alert

---

### ROOM EXPERIENCE (Test in the room)

- [ ] **10. Content sharing (HDMI ingest) works from a laptop**
  - Test: Plug in a laptop via the room's HDMI/USB-C cable, confirm content appears in meeting
  - Common fail: Cable worn out, wrong adapter, or BIOS USB setting changed post-update

- [ ] **11. Front-of-room display shows meeting info correctly**
  - Check: Room calendar tile shows next meeting name + time
  - If blank: Room mailbox permissions or Graph API consent may have lapsed

- [ ] **12. Microphone pickup covers the full table**
  - Test: Walk to farthest seat, speak at normal volume — confirm far-end hears clearly
  - Fix if failing: Check mic placement, enable 'Voice isolation' in Teams Rooms Pro settings

- [ ] **13. Touch console is responsive and not locked**
  - Check: Tap screen — responds within 1 second, no PIN lock screen blocking
  - If locked: MTR may have rebooted into Windows lock screen — needs local fix

---

### MANAGEMENT & MONITORING (Weekly, not just pre-meeting)

- [ ] **14. Teams Admin Center alerts are configured for this room**
  - Set up: TAC → Notifications & alerts → Device health rules
  - Minimum alerts: Offline >15 min, sign-in failure, peripheral disconnect
  - Why: Without alerts, you find out about failures from the CEO, not your dashboard

- [ ] **15. Teams Rooms Pro Management portal (if licensed) shows no open incidents**
  - Check: aka.ms/mtrp → Room health → Filter by severity: Critical/Important
  - Pro tip: Set weekly digest email to your team alias — 5 minutes every Monday

---

### Quick Reference: Failure → Fix Matrix

| Symptom | Most Likely Cause | First Fix |
|---|---|---|
| Room shows offline in TAC | Password expired on resource account | Reset password, disable expiry |
| Video freezes mid-call | Outdated camera firmware | Update via TAC |
| No audio from far end | UDP ports blocked | Open 3478–3481 UDP |
| Display doesn't wake | HDMI-CEC disabled | Enable CEC on display settings |
| Content share fails | Worn HDMI cable | Replace cable, test adapter |
| Console unresponsive | MTR crashed to Windows | Local reboot, check auto-login config |
