# The Teams Rooms Health Checklist: 15 Checks That Prevent 80% of Meeting Room Failures

# The Teams Rooms Health Checklist
## 15 Checks That Prevent 80% of Meeting Room Failures

*Complete this in under 15 minutes per room. Fix what you find. Stop firefighting.*

---

### WHY THIS EXISTS

The average mid-market company loses **4.2 meeting hours per employee per month** to room tech failures — bad audio, dropped calls, frozen screens, wrong room configs.

At a blended salary cost of €55/hr, a 200-person company is burning **€46,200/year** on preventable friction.

This checklist closes that gap.

---

## SECTION 1: Hardware & Connectivity (5 checks)

- [ ] **1. Camera firmware is current**
  - Check: Teams Admin Center → Devices → Cameras → Firmware version
  - Target: Must match latest release from manufacturer (Logitech, Poly, Yealink)
  - Fix if not: Schedule firmware push via TAC or manufacturer portal

- [ ] **2. Audio device shows no signal warnings**
  - Check: Join a test call. Confirm mic pickup at 3m distance without echo.
  - Target: No echo, no clipping, no dead zones
  - Fix if not: Check mic placement, disable AEC overrides, update speaker/mic firmware

- [ ] **3. Network: room device on dedicated VLAN or QoS-tagged**
  - Check: Network switch config or IT documentation
  - Target: Teams media traffic (UDP 3478–3481) prioritised
  - Fix if not: Raise with network team — single VLAN change, 30 minutes of work

- [ ] **4. Display HDMI/USB-C handshake is clean**
  - Check: Restart the MTR console. Does the display wake and show Teams UI within 20 seconds?
  - Target: < 20 sec wake, no blank screen on first join
  - Fix if not: Replace HDMI cable (most common cause), check display CEC settings

- [ ] **5. Content camera or front-of-room camera field of view covers all seats**
  - Check: Join from a remote device. Can you see every seat?
  - Target: 100% seat coverage
  - Fix if not: Adjust mount angle or upgrade to wider FOV camera

---

## SECTION 2: Software & Configuration (5 checks)

- [ ] **6. Teams Rooms app is on supported version**
  - Check: TAC → Devices → [Room Name] → Software version
  - Target: Within 1 major version of current release
  - Fix if not: Enable auto-update in TAC or push update manually

- [ ] **7. Room resource account has correct Exchange/M365 calendar permissions**
  - Check: Book a test meeting. Does it appear on the room console automatically?
  - Target: Meeting visible on console within 2 minutes of booking
  - Fix if not: Verify AutoAccept policy on resource mailbox in Exchange Admin Center

- [ ] **8. Teams Rooms Pro license assigned (not just Basic)**
  - Check: M365 Admin Center → Users → [Room Account] → Licenses
  - Target: Teams Rooms Pro (required for TAC monitoring, AI features, Copilot in meetings)
  - Fix if not: Assign Pro license — unlocks remote management and AI recap features

- [ ] **9. Coordinated meetings / proximity join enabled**
  - Check: TAC → Devices → Configuration profiles → Coordinated meetings
  - Target: Enabled, paired with Surface Hub or secondary devices if present
  - Fix if not: Enable in TAC config profile — zero hardware change needed

- [ ] **10. AI meeting recap / Intelligent recap enabled for the room**
  - Check: TAC → Meeting policies assigned to room account → Recording & transcription
  - Target: Transcription on, Intelligent recap enabled
  - Fix if not: Update meeting policy in TAC — takes effect within 1 hour

---

## SECTION 3: User Experience & Adoption (5 checks)

- [ ] **11. One-touch join works for Teams AND any required interop (Zoom, Google Meet)**
  - Check: Book a Zoom test meeting via the room calendar. Does one-touch join appear?
  - Target: One-touch join for all platforms your organisation uses
  - Fix if not: Enable Direct Guest Join in TAC → Devices → Configuration profiles

- [ ] **12. Room panel (if present) shows live availability and is synced**
  - Check: Walk up to room. Does panel show correct busy/free status?
  - Target: Real-time accuracy, < 1 min lag
  - Fix if not: Re-pair panel in TAC, verify resource account sync

- [ ] **13. Room has a visible, laminated quick-start card**
  - Check: Is there a physical card telling users how to start a meeting, share content, and get help?
  - Target: Yes — posted near the console or display
  - Fix if not: Print the template below and laminate it. 10 minutes.

  **Quick-Start Card Template:**
  ```
  [ROOM NAME] — How to meet
  1. Walk in → meeting appears on screen → tap JOIN
  2. Share content → tap Share on console or cable on table
  3. Problem? Call IT: [number] or Teams chat: [alias]
  ```

- [ ] **14. Usage data is being captured in TAC or Power BI**
  - Check: TAC → Analytics & reports → Usage → Teams Rooms
  - Target: At least 30 days of usage history visible
  - Fix if not: Ensure Pro license active (requirement) and report export enabled

- [ ] **15. Room is included in monthly health alert digest**
  - Check: TAC → Notifications → Alert rules — is this room in scope?
  - Target: Alerts enabled for offline device, poor call quality, peripheral disconnect
  - Fix if not: Add room to alert rule in TAC → Notifications → Manage alert rules
